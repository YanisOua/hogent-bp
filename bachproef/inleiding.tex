\chapter{\IfLanguageName{dutch}{Inleiding}{Introduction}}
\label{ch:inleiding}

De digitale agency-sector staat onder constante druk om de "time-to-market" van webapplicaties te verkorten. Bij Ballistix Digital is de initiële projectopstart momenteel een arbeidsintensief proces. Hoewel generatieve AI nieuwe mogelijkheden biedt, blijft de integratie van deze tools binnen een specifieke huisstijl een onontgonnen gebied.

\section{\IfLanguageName{dutch}{Probleemstelling}{Problem Statement}}
\label{sec:probleemstelling}
De wildgroei aan AI-generatietools biedt snelheid, maar mist vaak de context van bedrijfsspecifieke componentenbibliotheken. Dit leidt tot een "inconsistentiekloof": code wijkt af van de standaarden, wat resulteert in een hoge \textit{correction overhead}. Er is momenteel onduidelijkheid over welke tool of methode de meest "correcte" output levert voor een agency-specifieke workflow.

\section{\IfLanguageName{dutch}{Onderzoeksvraag}{Research question}}
\label{sec:onderzoeksvraag}
De centrale onderzoeksvraag van deze bachelorproef luidt: 
\textit{Welke AI-geassisteerde workflow levert de meest kwalitatieve en consistente projectstart op voor frontend-applicaties binnen een specifieke agency-huisstijl?}

Deelvragen:
\begin{itemize}
  \item Hoe verhouden algemene LLM's zich tot gespecialiseerde tools en een context-geoptimaliseerde setup (PoC via MCP) wat betreft consistentie?
  \item Welke invloed heeft het type input (visueel, functioneel of technisch) op de nauwkeurigheid?
  \item Bij welke tool is de \textit{correction overhead} het kleinst bij integratie in een bestaand template?
\end{itemize}

\section{\IfLanguageName{dutch}{Onderzoeksdoelstelling}{Research objective}}
\label{sec:onderzoeksdoelstelling}
Het doel is een vergelijkende studie tussen diverse AI-methoden. Er wordt een Proof-of-Concept (PoC) ontwikkeld op basis van het Model Context Protocol (MCP) om te testen of specifieke context de kwaliteit verhoogt.
